# Structure (profile: example-acme)

> 예시(가상) 프로필. 실데이터 아님.
>
> 회의의 **형태**를 정의 — 섹션 순서·카테고리·Action 그룹·제목 규칙·타입별 작성 규칙.
> 엔진은 형태를 강제하지 않는다. 이 파일이 있으면 그대로 쓰고, 없으면(profile=null) 녹취서 추론 후 확인.
>
> Acme = 가상 사내 단일 제품팀. "Smart Parking Pilot(SPP)" 운영. 다조직 컨소시엄 아님 —
> 운영/컨소시엄 회의록(안건·크로스조직 Action·증상 제목)과 **형태가 완전히 다르다.**

## 회의 카테고리
<!-- 이 조직이 실제로 하는 회의 종류. config.categories 키와 연결. -->
- standup — 주 3회(월·수·금) 스탠드업. 5줄 이내 초간결. 안건/Action 섹션 없음
- design-review — 격주 디자인 리뷰. 제안안 검토 → 결정/보류
- 1on1 — 주간 1:1. 공개 회의록 아님(개인 노트). 매니저 ↔ 팀원

## 카테고리별 섹션 순서
<!-- 각 카테고리 본문 섹션을 어떤 순서로. 운영회의의 "개요 → 안건별 논의 → Action Items"와 무관. -->
- standup: `진행 중` → `블로커` → `오늘 목표`
  - 안건 섹션 없음 / Action 그룹화 없음 / 5줄 이내
- design-review: `리뷰 대상` → `제안안` → `피드백` → `결정/보류(근거·대안·트레이드오프)` → `후속`
- 1on1: `근황` → `논의 주제` → `합의 사항`
  - 공개 회의록 아님 / Action 그룹화 없음 / 민감 내용 별도 분리

## Action Items 그룹 방식
<!-- 어떻게 묶나. 단일 조직 → 조직별 그룹 없음. -->
- **담당자별** — 사람 이름으로 묶음 (조직 라벨 (Acme)/(Beta) 류 없음, 단일 조직이므로)
  - design-review에서만 그룹화. 형식: `### 담당자` 아래 체크박스 항목
- standup / 1on1: Action 그룹화 없음 — 필요 시 문장 안에 인라인 표기

## 안건 제목 규칙
<!-- 운영회의의 "증상 기반·정량 마커"가 아니라 기획·리뷰형 제목. -->
- 안건명 그대로 / 주제 키워드
  - 예: "정산 화면 리디자인", "온보딩 플로우 3단계 단축"
  - 증상·장애·정량 마커(오류율·다운타임) 기반 제목 사용 안 함 — 운영 이슈 회의 아님

## 타입별 작성 규칙 (선택)
<!-- 이 회의 타입에만 적용되는 특수 규칙. -->
- **standup**: 5줄 이내 초간결. 수식어·배경 설명 생략. 블로커 없으면 "블로커: 없음" 한 줄
- **design-review**: 디자인 결정은 **근거 + 대안 + 트레이드오프** 3요소 명시
  - 예: "탭 → 아코디언 결정 / 근거: 모바일 세로공간 / 대안: 모달 / 트레이드오프: 한 번에 1개만 노출"
  - 보류 항목은 보류 사유 + 재논의 조건 명시
- **1on1**: 비공개·개인 노트. 민감/평가/처우 관련 내용은 본문과 분리해 별도 보관, 공유 산출물에 포함 금지

## 인라인 주석 마커 (`(*...)`)
<!-- 녹취록 원문에 남긴 사후 코멘트를 회의록 섹션으로 승격. 문법: `(*키워드_짧은설명)`, `*` 뒤 공백 허용. -->
- **2종 구분:**
  - **내용형** (`(*확인 필요: 정산 API 응답 스펙 — Platform팀)`) → 괄호 내용이 페이로드.
  - **태그형** (`(*정리)`, `(*인사이트)`) → 페이로드는 **붙은 발언** → (마커, host 발언) 쌍으로 추출.
- **키워드 → 섹션 라우팅** (기존 골격에만 — 신규 섹션 신설 금지):

| 마커 키워드 | 승격 위치 |
|---|---|
| `to-do` | design-review `후속` (담당자별) |
| `확인 필요` | `후속` — 답할 팀 병기 (예: Platform/Devices) |
| `인사이트` / `방향성` | `피드백` 또는 `결정/보류` |
| `정리` | `결정/보류` 앵커 |

- **담당자 접미:** `박상호D`처럼 이름+역할약어 → contacts.md 매핑. 없으면 `담당자 미지정` 플래그 — default 배정 금지.
- **미분류 마커:** 초안 하단 `### 미분류 마커` 블록에 노출. 불변식: 원문 마커 수 == 초안 반영 수.
- **provenance:** 사후 주석 ≠ 회의 발언 → 발언처럼 렌더 금지, 검토 주석으로 표기.
- **산출물 clean:** 공유본엔 raw `(*...)` 미포함.
