# Conventions (profile: example-acme)

> 예시(가상) 프로필. 실데이터 아님 — 새 프로필 작성 시 형태 참고용.
>
> 조직별 작성 관례. 엔진 writing-principles를 이 조직 맥락으로 특화.
>
> Acme = 가상 사내 단일 제품팀("Smart Parking Pilot(SPP)"). 다조직 컨소시엄 아님 —
> 내부 팀(Platform / Devices / QA)으로 나뉘고, Action은 **담당자(사람)별**로 묶는다.

## 문체
<!-- config.locale.business_style 보강. 이 조직 특유 표현·금지어·순화 규칙 -->
- 개조식 기본 — 명사 종결, 화살표(→) 적극 사용
  - 예: "정산 화면 리디자인 → 탭에서 아코디언으로 전환"
- 화자 표기는 **사람 이름** 기준 (예: 박상호, 정우진) — 조직 라벨 (Acme)/(Beta) 류 없음(단일 조직)
- 디자인 결정: [결정] / [보류] 표기, 보류는 재논의 조건 명시
- 금지: "~합니다/~됩니다" 체 (개조식에선 "~함/~필요/~실정")
- "및" 남용 금지 → 콤마(,) 또는 슬래시(/)

## 채널 관례
<!-- 카테고리별 공유 채널 세부(팀챗 채널명, 메일 제목 규칙 등). config.categories 보강 -->
- standup: 팀챗 채널 #spp-standup 에 5줄 이내 짧게 공유 (표 금지, Action 그룹 없음 — 인라인)
- design-review: 문서(Canvas/MD)로 공유 / 제목 `[SPP 디자인리뷰] 주제 — YYYY-MM-DD` / 내부 공유라 메일은 필요 시만
- 1on1: 비공개 개인 노트 — 공유 채널 없음, 매니저 ↔ 팀원 개별 보관

## Action Items 그룹 순서
<!-- 담당자(사람)별 그룹을 어떤 순서로 나열하는지. 조직별 그룹 없음 — 단일 조직 -->
- **담당자별** — `### 담당자 이름` 아래 체크박스 항목 (design-review에서만 그룹화)
- 나열 순서: 회의에서 언급된 순서 → 동순위면 이름 가나다순
- standup / 1on1: Action 그룹화 없음 — 필요 시 문장 안에 인라인 표기

## 호칭·직급
<!-- 매체별 직책 표기 외 조직 특유 호칭 규칙 -->

- 메일 받는사람(design-review 한정): "OOO PM님" / "OOO 리드님"
- 팀챗: 이름 + 님 (예: 박상호님)
